How this dashboard is wired

Live mode (badge: LIVE): this page is served by dash_server.py, which reads tier_table.csv + results.json, computes the stress index from calibration.py, runs jobs, and proxies chat to Claude with the key kept server-side.

Refresh followed universe → runs run_all.py (8-stage pipeline: discovery screen → TAM bridge → macro stress → Monte Carlo → leveraged → growth-regime → stress tests → dashboards), then this page reloads the numbers.

Discover candidates → runs the discovery/agent.py stage plan with its warm-start diff. The full discover→narrow→TAM flow keeps its human checkpoints by design — judgment inputs (TAM, archetypes) are confirmed by a person before the model prices them.

Chat → POST /ask: Claude answers as the model’s analyst over the live table, macro and matrices. Without a server key, the dashboard falls back to built-in offline answers from the embedded snapshot.

Production path: schedule the followed-universe run (cron/systemd), add a FRED/market-data ingestion module for the macro inputs, put auth + HTTPS in front, and pin the chat to the published tail definition.
